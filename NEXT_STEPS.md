# Next Steps After v1.5.8 Autonomy/Nesting

## 1. Long-run auto-mode stability

Run `Brain: auto` on Linux Mint Cinnamon 22 for at least an overnight session. Watch survival pressure, active mode changes, live structures, archive growth, nested phenomena count, replay proof length, and IndexedDB save size.

## 2. Promote nested phenomena through shadow testing

The new nested layer is virtual and safe. The next major upgrade is a controlled promotion path:

```text
virtual nested phenomenon → candidate superstructure → shadow test → commit/reject → archive/fossil
```

No nested phenomenon should become live until it passes the same leech/cascade/objective guardrails as ordinary candidates.

## 3. Full replay rebuild

Current replay verifies the retained hash chain. Build the stronger test: reconstruct a world from seed + replay events, then compare the reconstructed final digest against the exported proof.

## 4. Executable child-world forks

Persistent child worlds are currently records. Add UI actions to export a child as a `.spectreverse.child.json` seed or load it as a new active local save with its own replay proof.

## 5. Vector epoch ledger

Persist structure/nest/world vectors over time:

```text
structure vector → nest vector → world vector → epoch vector
```

Use vector deltas as an objective term so the system can select for genuine phase-space movement instead of merely accumulating structures.

## 6. Living Word packet authoring

Keep self-modification packet-based. Add an in-browser packet editor for operator tweaks, atlas seeds, mutation rules, objective profiles, failed-pattern quarantine, and promotion thresholds.

## 7. Save Garden transfer tests

Run cross-save transfer explicitly: take top structures/nests from Save A, shadow-test them in Save B, and label them local / portable / universal / poison / dormant.

## 8. Multi-worker population phase

Only after replay rebuild + nested promotion are stable, split child worlds into independent Workers with budget limits, local objective profiles, save slots, and quarantine before absorption.

## 9. Acceleration phase

Move heavy scorer/search kernels to WASM/WebGPU only after safety gates are stable. Acceleration should amplify validated selection pressure, not just generate more candidates.
